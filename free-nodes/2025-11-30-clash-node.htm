引言：签到不再靠记忆，让自动化帮你搞定

在众多网络自动化工具中，Quantumult X凭借其强大的代理能力与灵活的脚本支持，逐渐成为iOS用户的首选。不仅能科学上网，还可以配合脚本完成诸如“自动签到”“自动领券”“定时推送”等高级操作，堪称数字生活的“隐形助手”。

一、什么是Quantumult X？为何适用于京东签到？

1. 简要介绍

Quantumult X 是一款专为 iOS 平台设计的网络代理工具，支持 HTTP(S)、Shadowsocks、Vmess、Trojan 等多种协议，同时集成了脚本引擎，可用于捕获网络请求、修改数据包，甚至模拟用户操作。这一特性正是自动化京东签到的核心所在。

2. 核心优势

  • 脚本自定义能力强：支持 JavaScript，运行灵活。

  • 分流控制精准：能控制请求是否走代理、是否触发脚本。

  • 资源丰富：已有大量开发者贡献签到/任务类脚本。

  • 兼容性好：与Surge等规则格式通用，便于迁移或套用资源。

二、京东签到的价值与为什么要自动化

1. 京东签到的意义

京东每日签到通常能获得如下福利：

  • 京豆（可抵现金）

  • 无门槛优惠券

  • 游戏抽奖机会

  • 满减红包与专属任务

2. 为什么不手动而要自动化？

  • 重复性高：每日一次，节假日也不能断。

  • 容易遗忘：早晨忘了点，晚上再想起来奖励可能已过期。

  • 性价比高：简单设置一次，持续获取奖励。

  • 支持多人：可帮多个账号签到，提升收益效率。

自动化不仅节省操作成本，更符合精益生活的理念。

三、Quantumult X 安装与基本准备

1. 安装 Quantumult X

  • 打开 iOS App Store，搜索 Quantumult X

  • 付费购买（一次性解锁或订阅）

  • 下载并安装后，授权必要权限（网络、通知）

注意：Quantumult X 是苹果商店中的付费应用，价格约在 30 元人民币上下，购买后即可终身使用基本功能。

四、构建京东签到自动化体系

1. 获取京东签到脚本

市面上已有开发者制作好的“京东签到合集”脚本，通常托管于 GitHub 或 jsDelivr 等CDN服务中。常见脚本包含：

  • 每日签到

  • 京豆领取

  • 工具类签到任务（如京东金融、京东极速版）

脚本来源推荐：

  • @NobyDa 的京东签到合集

  • @chavyleung 的签到库

  • Quantumult X 脚本分享站点

2. 添加远程配置

打开 Quantumult X：

  • 进入【配置】 → 【添加配置】

  • 选择“远程配置”

  • 输入脚本链接（例如：https://example.com/jd_checkin.conf）

  • 导入并启用配置

此配置文件包含：

  • 远程脚本地址

  • 匹配京东请求的分流规则

  • 脚本触发机制

五、获取京东Cookie的正确方式

脚本通常需要用户的 Cookie 才能模拟签到。以下是获取方式：

方法一：使用 Quantumult X 监听请求

  1. 打开 Quantumult X，开启“MitM”（中间人代理）并导入信任证书。

  2. 将 iOS 设备的 WiFi 代理指向 Quantumult X。

  3. 在 Safari 中登录京东官网。

  4. Quantumult X 会捕获登录请求并自动提取 Cookie。

  5. 你会在日志中看到如 JD_COOKIE 的字样，即表示成功。

方法二：浏览器开发者工具（适用于电脑端）

  1. 使用 Chrome 或 Edge 打开 https://m.jd.com

  2. 登录你的京东账号

  3. 右键页面 → 检查 → 切换到“Network”标签

  4. 刷新页面，找到请求如 home 或 client.action

  5. 查看“请求头”（Headers）中的 Cookie 字段

  6. 复制粘贴至脚本配置中

六、脚本运行与签到流程

1. 自动执行脚本

多数签到脚本都设定为 定时触发，你只需确保 Quantumult X 正常运行，它会在后台定时请求。

可在配置文件中看到类似：

csharp
[task_local]
0 8 * * * https://example.com/jd_checkin.js, tag=京东签到

上述含义是：每天早上 8 点自动运行一次签到脚本。

2. 手动触发脚本

  • Quantumult X → 【脚本】→ 找到签到脚本 → 点击执行

  • 控制台会显示执行结果和返回内容

3. 查看签到结果

  • 可在控制台或通知中查看签到是否成功

  • 登陆京东查看京豆数量是否增加

七、常见问题解答

Q1：签到脚本失效了怎么办？

  • 检查脚本地址是否被维护

  • 更新至最新版本的脚本

  • 删除旧 Cookie，重新抓取一次

Q2：为什么脚本无法运行？

  • 检查 MitM 配置是否开启并添加了 *.jd.com 域名

  • 检查时间规则是否生效

  • 检查是否授权网络代理和后台运行权限

Q3：是否能多个账号签到？

是的，部分脚本支持 多账号 Cookie 的粘贴，只需以 & 符号连接多个账号的 Cookie 即可。

八、进阶操作建议

1. 与通知系统结合

开启 Quantumult X 的通知推送，签到成功/失败后自动推送结果，第一时间掌握签到状态。

2. 与 Telegram Bot 或 Bark 集成

高级用户可结合 PushPlus、Bark、TG Bot 等服务，将签到信息推送到你的设备或聊天频道中。

3. 使用插件脚本整合更多任务

许多脚本开发者将多个京东功能合并到一个 JS 脚本内，可通过配置参数自由启用签到、抽奖、领券等功能。

FAQ

Quantumult X 的自动策略组如何工作？
Quantumult X 的自动策略组可根据延迟、可用性或测速结果自动选择最优节点，用户无需手动切换。可在策略组设置中开启“延迟优先”或“可用性优先”模式以优化体验。
Hysteria2 的多路复用功能有哪些好处？
多路复用允许多个短连接共享同一 QUIC 通道，减少握手延迟和资源消耗。在高并发或频繁短连接场景下，能显著提升吞吐量和稳定性，尤其适合游戏或即时通讯应用。
VLESS + XTLS 如何降低移动网络延迟？
XTLS 通过减少握手次数和加密开销降低延迟。结合策略组和多节点切换，可在高延迟或不稳定的移动网络环境中保持连接稳定，提升网页浏览、视频和游戏体验流畅度。
Mihomo 是什么？
Mihomo 是 Clash Premium 的开源替代项目，兼容原版 Clash 配置文件并扩展了部分新功能。它性能优异、内存占用低，适用于希望自定义构建代理环境的高级用户。
VMess 的安全性是否优于 Shadowsocks？
VMess 拥有动态验证机制和更复杂的握手协议，相较于静态密钥的 Shadowsocks 安全性更高。它能有效抵御重放攻击和 DPI 检测，更适合长期使用。
VLESS + XTLS 模式在移动网络下表现如何？
XTLS 降低加密开销，减少握手次数，适合高延迟或不稳定的移动网络。结合策略组和多出口节点，可提升访问速度和连接稳定性，确保用户体验流畅。
VMess 的 gRPC 传输适合移动网络吗？
gRPC 传输支持长连接和多路复用，减少握手次数，降低延迟。对于移动网络和高延迟环境，它能提供稳定和高速的访问体验，同时降低掉线和延迟波动风险。
QuantumultX 支持哪些定制化分流策略？
QuantumultX 支持域名、IP、端口、协议以及用户自定义脚本分流。结合策略组使用，可实现广告屏蔽、视频加速、游戏优化等功能，提高访问速度和个性化网络体验。
Clash 节点测速如何提高访问效率？
Clash 会周期性检测节点延迟和丢包率，根据测试结果自动选择最优节点。结合策略组和分流规则，用户可体验更快的网页加载、视频播放和游戏连接速度，提高整体网络效率。
Surfboard 的策略组可以根据延迟自动切换节点吗？
支持。Surfboard 可定期测速节点延迟，并结合策略组自动选择延迟最低的节点。这样用户无需手动切换节点即可保持高速访问，提高网络体验和稳定性。
